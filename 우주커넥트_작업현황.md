## 🔧 크롤러 동작 방식

### Railway 서버 API
- `GET /` - 서버 상태 확인
- `GET /status` - 크롤링 상태 확인 (running, last_run, last_result)
- `POST /crawl` - 크롤링 실행 (헤더: x-crawl-secret: woozoo-crawl-secret)

### 크롤링 흐름
1. Supabase `plan_mapping`에서 is_active=true 요금제 목록 가져오기
2. `model_mapping`에서 SKT모델명 → 모델코드 매핑 로드
3. 요금제별 × 가입유형별(MNP/CHG/NEW) SKT 사이트 크롤링
4. 신규 SKT 모델명 → model_mapping에 자동 등록 (is_active=false)
5. 매핑된 모델만 support_amount에 upsert 저장 (번호이동 기준 최고금액)

### index.html CRAWLER_URL
```javascript
const CRAWLER_URL = 'https://web-production-3098b.up.railway.app';
const CRAWL_SECRET = 'woozoo-crawl-secret';
```

## ⚠️ 새 대화에서 이어받을 때 주의사항

1. **index.html은 항상 최신 파일을 업로드**해서 작업할 것
2. **브라우저 번역기 끄기** - 번역기가 URL, 변수명까지 한글로 번역해서 문제 발생했음
   - Railway URL: `web-production-3098b` 가 `웹프로덕션-3098b`로 번역됨
   - railway.json key값들이 한글로 번역되어 Railway가 인식 못함
3. **Railway 서버 포트**: 8080 고정 (railway.json에서 --port 8080 명시)
4. **크롤링 실패 원인**: Vercel에 최신 index.html이 배포됐는지 확인 필요
5. **크롤러는 Playwright 기반**이며 Railway 서버에서 실행됨
